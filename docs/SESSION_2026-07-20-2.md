## 1. 퍼즐 난이도 상향 (함정 재정의 + 화살 재배치)

- **함정의 정의**를 코드 기준으로 정립: 벽(`freeSteps=0`, 실수 안 남) / 공간 함정
  (`freeSteps≥3`, 멀리서 막힘) / 시간 함정(순서 강제 = 분기수).
- **실측 진단**(출하 bank.json 775판): 밀도 97%·2셀 41%·함정 9.3%·분기 19 → 너무 쉬움.
- **해결 3레버** — `tools/atlas/relayout_difficulty.py`(신규):
  ① 밀도 상한(리페어 중단) ② LONGER 길이믹스 ③ **체인 재배향**(머리를 다른 화살을
  가장 많이 가로지르는 쪽으로 → 공간 함정 + 순서 강제 동시).
- **결과**: 랭크 기반 커브(초반 함정 6.6%·분기 15 → 후반 20%·7.3), **775판 전량 풀림**.
  grid/판 크기 불변, 화살 배치만 재생성. 상세: `docs/DIFFICULTY_PLAN.md`.
- ⚠️ **뱅크 재생성 시 `relayout_difficulty.py`까지 돌려야 난이도 유지**(HANDOFF에 경고).
- 결정 A(개수 스펙 폐기→구성+심도) · B(만들며 측정 위임) 확정 반영.

## 2. 플레이 입력·제스처

- **드래그 시 화살 오발사 방지**: 발사를 `onTapDown`→`onTapUp` + 터치 슬롭 검사
  (`LineComponent._tapSlop`). 드래그는 발사 안 됨.
- **보드 팬/줌을 Flutter로 이관**: Flame `ScaleCallbacks`는 실기에서 한 손가락 팬을
  못 잡음 → `GameWidget`을 `GestureDetector`+`Transform`으로 감싸 처리(1손가락 팬,
  2손가락 줌). 탭은 Flame이 받음(Transform 좌표 역변환). Flame 팬/줌 코드 제거.
  '맞춤 보기' 버튼·새 보드 진입 시 뷰 리셋.
- **우측하단 +/- 줌 버튼 삭제**(`_ZoomControls` 제거). 핀치 줌·맞춤버튼 유지.
