# 월드맵 재기획 — 4단 줌 + 여권 스탬프

작성: 2026-07-20 · 상태: **기획안 (구현 전, 결정 필요 항목 있음)**

기존 월드맵은 전세계 점격자 한 장에 3색(진행중/클리어/잠금)뿐이었다. 캠페인이
면적 오름차순이라 초반 수십 라운드가 미세 섬나라여서 **화면이 사실상 변하지 않았고**,
성취 엔진으로 기능하지 못했다.

---

## 0. 설계 원칙

기획서(`PUZZLE_BANK_STRATEGY.md`)의 확정 사항을 그대로 따른다.

- **성취감의 원천은 지도다.** 실루엣은 포장이고 게임성이 아니다
- **복잡한 걸 요구하지 않는다.** 지도는 보는 것이고, 읽거나 계산하게 하지 않는다
- **한 판마다 눈에 띄게 변해야 한다.** 라운드 완주까지 기다리게 하면 피드백이 너무 멀다

---

## 1. 4단 줌 구조

```
전세계  →  대륙  →  국가  →  도시(보드)
World      Continent  Country   City
```

한 방향으로만 파고들고, 뒤로 나오면 한 단계씩 넓어진다. 각 단계는 **자기 층의
수집 현황을 보여주는 화면**이지 지도 그 자체가 아니다.

### 1.1 전세계 — 대륙 스탬프 6칸

- 대륙 6개를 카드/영역으로 배치. 각 칸에 **대륙 스탬프**(획득 시 컬러, 미획득 시 흐림)
- 하단: `대륙 스탬프 2 / 6`, 최종 칭호 진행도
- 대륙을 탭 → 대륙 화면

### 1.2 대륙 — 비자 스탬프 N칸

- 그 대륙의 국가들만 그린 점격자 지도(전세계 격자에서 해당 대륙만 크롭)
- 국가별 상태: **미개방 / 진행중(부분 채색) / 비자 획득(완전 채색 + 스탬프 도장)**
- 상단: `대한민국 외 12개국 · 비자 13 / 52`
- 국가를 탭 → 국가 화면

### 1.3 국가 — 라운드 진행

- 국가 실루엣 위에 스테이지 핀(도시들 + 국가 피날레). `CampaignCountry.pins` 재사용
- 클리어한 핀은 채워지고, 다음 핀이 강조됨
- 전부 클리어 시 **비자 스탬프가 실루엣 위에 찍히는 연출**
- 핀을 탭 → 보드(플레이)

### 1.4 도시 — 기존 게임 화면

변경 없음.

## 2. 스탬프 · 칭호

### 2.1 국가 비자 스탬프

| 조건 | 획득 |
|---|---|
| 도시가 있는 국가 | 도시 스테이지 전부 + 국가 스테이지 클리어 |
| 도시가 없는 국가 | path 스테이지 + 국가 스테이지 클리어 |

> ⚠️ **도시 없는 78개국의 path 스테이지는 아직 미생성이다.** 현재는 국가 스테이지
> 1개뿐이라 "1판 = 비자"가 되어 다른 국가와 무게가 맞지 않는다.
> path 보드 78개(사각형 마스크, 화살 80 이상)를 새로 생성해 넣어야 이 규칙이 성립한다.
> 기존 문제 자산은 건드리지 않고 **추가 생성**한다.

### 2.2 대륙 스탬프 + 칭호

해당 대륙의 **모든 국가 비자**를 모으면 획득.

| 대륙 | 국가 수 | 칭호(안) | 근거 |
|---|---|---|---|
| Europe | 50 | **구대륙 순례자** | 성지·도시국가 밀집, 순례 전통 |
| Asia | 52 | **실크로드 대상(隊商)** | 육상 교역로의 상징 |
| Africa | 55 | **사바나 종단자** | 대륙 종단이 곧 난이도 |
| North America | 31 | **신대륙 개척자** | 프런티어 서사 |
| Oceania | 15 | **남십자성 항해사** | 태평양 도서 항해 |
| South America | 13 | **안데스 등정자** | 대륙을 관통하는 산맥 |

**대륙은 6개다.** `campaign.json`의 `continent`에는 Natural Earth 원본의
`Seven seas (open ocean)`이 그대로 남아 있는데, 이건 대륙이 아니라 **어느 대륙에도
붙지 않는 원양 영토를 담아둔 잔여 버킷**이다("칠대양"이라는 7개짜리 묶음이 아니다).
들어 있는 4곳은 전부 인도양 섬이므로 실제 소속으로 흡수한다:

| 국가 | 편입 | 근거 |
|---|---|---|
| 세이셸 · 모리셔스 | Africa | 아프리카연합 회원국 |
| 영국령 인도양 지역 | Africa | 인도양 · 아프리카 권역 |
| 허드 맥도널드 제도 | Oceania | 호주 외부영토 |

`build_bank.py`가 `continent`를 실을 때 이 매핑을 적용한다.

- 칭호는 홈 화면 로고 아래 / 설정 상단에 표시 (한 줄, 읽기를 요구하지 않음)
- **여러 개 보유 시 가장 최근 것 표시 + 전체는 여권 화면에서 확인**

### 2.3 최종 칭호

6개 대륙 스탬프 전부 → **Grand Tourist / 그랜드 투어리스트**

## 4. ⚠️ 결정이 필요한 것 — 진행 순서와 대륙 완성의 충돌

**이게 이 기획의 핵심 쟁점이다.**

캠페인 순서는 `rank` 1~216(바티칸→러시아)으로 확정돼 있고, 이 순서는 **대륙을
무작위로 넘나든다.** 그 결과:

- 어떤 대륙도 캠페인 후반까지 완성되지 않는다
- **대륙 스탬프 7개가 사실상 전부 막판에 몰린다**
- 중간 목표로 기능하지 못한다 (기획서 6장의 "대륙 단위 중간 목표"와 어긋남)

선택지:

| 안 | 내용 | 대가 |
|---|---|---|
| **A. 순서 유지** | 대륙 스탬프는 순수 장기 목표. 중간 보상은 비자 스탬프가 담당 | 대륙 칭호가 엔드게임 전용이 됨 |
| **B. 대륙 선택제** | 대륙을 고르면 그 안에서 rank 순으로 진행. 잠금을 "대륙별 진도"로 변경 | 선형 `unlocked` 하나로 못 함 — 진행도 모델 변경 필요 |
| **C. 혼합** | 기본은 rank 순 추천, 다만 **이미 연 대륙 안에서는 자유 선택** | 구현·설명 비용 중간 |

현재 진행도는 `Progress.unlocked` **정수 하나**다. B/C를 택하면 국가별 클리어 상태를
저장하는 구조로 바꿔야 한다(예: 클리어한 스테이지 인덱스 집합).

**A가 아니면 진행도 저장 형식이 바뀌므로, 이 결정 전에는 구현에 들어가지 않는다.**

## 5. 구현 순서 (결정 후)

1. `build_bank.py`에 `continent` 싣기 (`campaign.json`에서 조인)
2. 진행도 모델 확정 (4장 결정에 따라)
3. 국가 화면 — 실루엣 + 스테이지 핀 (가장 자주 보게 될 화면)
4. 대륙 화면 — 대륙 크롭 지도 + 비자 카운터
5. 전세계 화면 — 대륙 스탬프 7칸
6. 스탬프 획득 연출 + 칭호 표시
7. path 보드 78개 생성 → 도시 없는 국가 규칙 성립
