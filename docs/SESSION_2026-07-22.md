## 1. 플레이 화면 — 팬/줌/탈출 (PR #15, `6ae93c1`)

### 팬 클램프를 격자 기준으로
- 뷰포트(child) 가장자리로 막고 있어서, `0.94` 여백과 레터박스만큼 격자가 안쪽에
  인셋된 걸 무시했다 → **레터박스 축에서 격자가 화면 중앙을 20~30% 넘어감.**
- `AtlasArrowsGame.boardRect`(격자 실측 rect)를 노출해 그 가장자리로 클램프.
- 실기 확인: 상·하·좌·우 전부 플레이 영역 중앙선에서 정확히 정지.

### 줌 상한을 보드마다 계산
- `maxScale = 8` 고정이 양쪽 다 틀렸다. bank.json 1,549판 실측 결과 fit 상태
  셀 크기가 **2.6dp(138열) ~ 9.8dp(27열)** 로 4배 차이.
  - 큰 보드: 8배로도 21dp → **탭 가능 최소 26dp에 미달**
  - 작은 보드: 8배면 78dp → 5열만 보임
- 규칙을 **"셀이 48dp가 될 때까지"** 하나로 바꿔 보드별로 계산(2~16배 클램프).
- 축소는 fit(1배)이 하한 — 기존과 동일.

### 탈출 화살이 화면 밖으로 (3단계로 고쳤다)
1. **렌더 표면을 전체화면으로.** 게임 표면이 헤더~부스터바 사이였고 `ClipRect`가
   걸려 있어, 광선을 아무리 늘려도 보드 위 여백에서 잘렸다. 표면을 본문 전체로
   넓히고 헤더·하트·부스터바를 **불투명하게 위에** 얹어 슬라이드-언더 구현.
   보드 fit·중심은 `setPlayInsets`로 넘긴 **보이는 영역** 기준(크롬 높이 실측).
2. **팬/줌 대응.** 목표가 '캔버스 끝'이었는데 팬하면 캔버스 끝이 화면 한가운데로
   들어온다(클램프가 격자 끝을 화면 중앙까지 허용하므로 최대 화면 절반). →
   `AtlasArrowsGame.visibleRect`(팬/줌 역변환)를 목표로 변경.
3. **수축 제거.** 광선이 '머리가 가장자리를 넘을 만큼'만 길어서, 머리가 경로 끝에
   닿으면 이동이 멈추고 몸통만 제자리에서 줄었다. 광선에 몸통 길이를 더하고
   제거 시점을 꼬리→머리 기준으로 이동(사라지는 타이밍은 동일).
- 판별 검증: 오른쪽으로 팬해 캔버스 좌측 끝을 화면 중앙(x=677)에 둔 상태에서
  좌향 화살 발사 → **369px 전체 길이 유지한 채 x=0까지 이동 후 화면 밖으로.**

### 격자 여백
- 화면 비율(%)이 아니라 **고정 dp**로 변경. %면 화면이 클수록 여백만 비례해
  커져 격자가 상대적으로 작아진다.
- 값은 "꽉 찬 상태의 0.9배"(`_marginX = 22dp`). 실측 좌우 90/96px.
- ⚠️ **설정한 여백과 보이는 여백이 8px 다르다** — 화살이 보드 사각형 가장자리까지
  닿지 않는다. 이 값 만질 땐 실측 필수.

### 기타
- 화살 머리 1.2배(밑변 0.5 → 0.6셀 = 축 굵기의 3배).
- **빌드 블로커 수정**: `sentry_flutter 8.x`가 Kotlin `languageVersion "1.6"`을
  하드코딩해 Kotlin 2.3에서 컴파일 실패 → `android/build.gradle.kts`에서 플러그인
  모듈만 2.0으로 되돌림. `pubspec.lock` 누락 항목 8개도 보충.

## 2. 워드마크 — Figma 스펙 확정

- 폰트 `Outfit ExtraBold` → **`Outfit Bold`**, 크기 112,
  **ATLAS +6% / ARROWS −5%**, 줄간격은 원본 실측(캡높이 83에 96)에서 역산.
- 사용자가 처음 준 값은 "ATLAS −6%"였는데, 그걸로 렌더하면 첨부 PNG가 재현되지
  않았다(비율 0.72 vs 0.83). 나란히 렌더해 비교 → **부호가 반대**임을 확인.
  +6%로 하면 모든 측정치가 PNG와 0.2% 이내 일치.
- ⛔ **두 줄을 폭 맞추지 말 것.** ATLAS를 늘려 두 S를 정렬하면 좌우대칭 사다리꼴이
  직사각형이 된다 — 그 실루엣이 마크의 정체성이다. 코드·생성기 양쪽에 주석 고정.
- 로딩 플레이트용 PNG 에셋(`tools/atlas/build_wordmark.py`)도 같은 스펙으로 재생성.
  기존 생성기는 두 줄을 **각각 1184px로 늘려서** 사다리꼴이 아예 없었다.
- **결정(사용자 확인)**: 폰트 렌더 유지. PNG 직접 사용은 다크모드에서 ARROWS가
  안 보이고(슬레이트 고정) 463→576px 확대로 뭉개져서.

## 3. 홈 화면

- 워드마크 세로중앙을 **화면 상단 ~ 맵 상단의 정중앙**에 배치. 실측 466px = 이상값
  466px 일치.
- 시작하기 버튼을 **자기 높이(58)만큼 위로**.
- 월드맵 도트 `inkFaint 0.45` → **`inkSoft 0.7`** (라이트에서 #CBCFD0 → #8A959B).

### ⚠️ 이 과정에서 한 번 크게 틀렸다 (재발 방지)
버튼만 올리라는 지시에 Column 구조를 통째로 바꿔 **맵을 324px 위로 옮겼다**
(CLAUDE.md §3 위반). 원인은 **맵이 Column의 유일한 가변(Expanded) 요소**라는 것 —
위아래 어느 박스를 건드려도 맵이 따라 움직인다. `git checkout`으로 원복 후,
워드마크와 CTA 둘 다 **`Transform.translate`로 자기만** 이동시켰다. 레이아웃
공간이 그대로라 맵은 1px도 안 움직인다.

또 하나: **`SafeArea`는 `padding`뿐 아니라 `viewPadding`까지 깎는다.** 안쪽에서
상태바 높이를 물으면 0이 돌아온다. "화면 전체 기준" 배치는 값을 **SafeArea 바깥에서**
잡아 내려야 한다. 이걸 몰라 두 번 헛빌드했다.

## 5. 플레이 화면 UI

- **헤더**: `STAGE N` → `AppText.headline 18 / w900`(홈 CTA와 동일).
  한국어 로케일이면 **`스테이지 N`**.
- **하트를 헤더 우측 끝으로.** Row → **Stack**으로 교체 — Row로는 뒤로가기 버튼과
  하트의 폭이 정확히 같아야만 중앙이 맞는데, 하트는 소모되며 폭이 변한다.
  Stack의 `Center`는 그와 무관하게 항상 화면 가로 중앙.
- **장소 칩**: 헤더 바로 아래 가운데 `도시 · 국가 + 국기`, `surface 62%` 반투명.
  레이아웃 공간을 먹지 않고 떠 있다(반투명 = 뒤 보드가 비쳐야 한다는 뜻으로 해석).
- 광고 영역 `_bannerSlot` 100 → **92** (io/stub 양쪽).
- 아이템 바 하단 패딩 10 → **30**(배너와의 간격 3배).
- **아이템 버튼 4개 전부 1.3배**(부스터 66×58→86×75, 유틸 58×58→75×75),
  아이콘도 1.3배, 라벨 **20 / w500 / inkSoft**.
- 개수 뱃지: 원 21 → **30**(16pt 숫자가 넘쳤다), 컬러 `ink → inkSoft`.
  뱃지는 메타데이터고 주인공은 아이콘인데 위계가 뒤집혀 있었다.
  `+`(구매 유도)는 accent 유지 — 재고 표시와 하는 일이 다르다.

## 6. 전역 정리

### 폰트 16 미만 전부 제거
타입 스케일 자체를 올렸다: `label 14→16`, `caption 12→16`. 개별 `fontSize:` 축소
지정 10곳(10.5, 9, 11, 13, 13.5, 15)을 삭제해 토큰을 따르게 했다.
`lib/` 전체에 16 미만 없음.

### 버튼 통일
세로 패딩이 **12·13·14·15·16·17·18** 7가지, 굵기가 **w600·w800·w900** 3가지로
흩어져 있었다. `kButtonPadV = 18` / `kButtonText = headline w800` 토큰
(`app/tokens/typography.dart`)으로 홈·온보딩·결과시트·실패시트·확인팝업·클리어바
전부 통일.

### 탭바
- 활성 알약이 **아이콘+텍스트 전체**를 감싸도록.
- 알약 **고정폭 68** — 내용물 크기로 잡혀 있어 `홈`(1글자)과 `상점`(2글자)의 폭이
  달랐다.
- 비활성 색 `inkFaint → inkSoft`.

### 하트 소진 바텀시트
- 하트 이미지 3개 삭제, 문구 → **`계속 플레이 하세요.`**
- 버튼 → **하트 아이콘 + `리필쿠폰(n)`**
- **무료 리필은 평생 1회.** `_freeRefillUsed`(스테이지마다 리셋되던 bool)를
  `Progress.refillCoupons`(영속)로 교체. 기존엔 스테이지를 넘기거나 재시작할
  때마다 무료 리필이 부활했다.
- 녹색(`c.success`) 버튼 → **`c.accent`**. 녹색은 완료를 뜻하는 색인데 충전
  버튼에 쓰여 있었다. 광고 버튼도 `accentSoft` → `accent`(쿠폰이 떨어지면 그게
  유일한 선택지라 주 버튼이어야 한다).

### 나가기 팝업 정렬
`showDialog`의 기본값 `useSafeArea: true` 때문에 **상태바를 뺀 영역의 중앙**에
놓여 있었다 → 박스가 아래로 밀려 제목이 화면 중앙에 왔다. `useSafeArea: false`.

### 효과음 음계
`comboIndex.clamp(0,7)`이라 8번째부터 최고음만 반복됐다. 삼각파로 교체해
`0→7→0→7…` 오르내린다.

### 치트 버튼
설정 화면 하단 **LOGAN 10탭 = ON / LAND 10탭 = OFF**(`Progress.cheatOn`, 영속).
켜면 플레이 화면 우상단에 `CLEAR` 버튼이 뜨고, 누르면 실제 클리어와 같은 경로로
스테이지를 클리어 처리한다(리빌·언락·광고 케이던스 모두 정상 동작).
