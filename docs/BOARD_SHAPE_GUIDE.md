# Z-Arrows 퍼즐보드 실루엣 아이데이션 가이드

> 이 문서를 그대로 GPT에 붙여넣고 실루엣 아이데이션을 요청하면 됩니다.
> 결과물은 파싱해서 바로 게임의 마스크(BoardMasks)로 사용합니다.

---

## 게임 컨텍스트

Z-Arrows는 화살표 선들이 얽힌 미로를 푸는 모바일 퍼즐이다. 보드는 격자 위 셀들의
집합(마스크)으로 정의되고, 그 안을 생성기가 화살표로 빽빽하게 채운다.
**너의 임무는 화살표 배치가 아니라 보드의 실루엣(외곽 형태)만 디자인하는 것이다.**
풀 수 있는지는 신경 쓸 필요 없다 — 엔진이 보장한다.

## 출력 형식 (반드시 준수)

각 실루엣을 ASCII 격자로 출력한다. `#` = 보드 셀, `.` = 빈 칸.
모든 행의 길이는 같아야 한다(직사각형 바운딩 박스).

```
name: heart
category: symbol
size: 12x13        # rows x cols
difficulty: boss   # normal | boss
.##...##.....      # ← 13열이면 13글자
############# 
(...12행...)
```

예시 — 하트 (12x13):

```
name: heart
category: symbol
size: 12x13
difficulty: boss
.###....###..
#####..#####.
#############
#############
#############
.###########.
..#########..
...#######...
....#####....
.....###.....
......#......
.............
```

## 하드 제약 (어기면 사용 불가)

1. **격자 크기**: 세로(rows) 10~18, 가로(cols) 8~13. 모바일 세로 화면이므로
   **세로 ≥ 가로** 권장 (비율 1:1 ~ 1.6:1).
2. **셀 수**: `#`의 총 개수 80~200개. (플레이 타임 1분 이상 확보 조건.
   normal은 80~130, boss는 130~200)
3. **연결성**: 모든 `#`은 상하좌우로 하나의 덩어리로 연결되어야 한다.
   떨어진 섬 금지. (대각선 연결은 연결로 치지 않는다)
4. **두께**: 형태의 대부분은 **2셀 이상 두께**여야 한다. 1셀 폭의 가는 줄기는
   전체의 20% 이하로만 (1셀 폭에는 직선 화살표밖에 못 들어가서 재미가 없다).
5. **디테일 한계**: 1셀보다 작은 디테일은 표현 불가. 곡선은 계단으로 근사된다.
   구멍(도넛 등)은 가능하지만 최소 2x2셀 이상이어야 시각적으로 보인다.

## 디자인 지향점

- **실루엣만으로 무엇인지 읽혀야 한다.** 게임 화면에서는 외곽선 없이 점(빈 칸)과
  화살표 밀도 차이로만 형태가 드러난다. 볼드하고 아이코닉하게.
- 좌우 대칭 형태가 잘 읽힌다 (비대칭은 동물/사물처럼 방향성 있는 소재에만).
- 챕터당 일반 레벨은 기하/유기 형태, **챕터 마지막(boss)은 그림 레벨**
  (클리어 후 "이 판은 ♥ 모양이었다" 연출용) — boss는 소재가 명확한 것.

## 요청 카테고리 (각 카테고리별로 다양하게)

| category | 예시 소재 |
|---|---|
| geometric | 원, 다이아몬드, 육각형, 십자, 도넛, 계단, 나선 프레임 |
| organic | 블롭, 구름, 물방울, 나뭇잎, 조약돌 |
| symbol | 하트, 별, 달, 번개, 음표, 물음표, 열쇠, 왕관 |
| animal | 토끼, 고양이, 새, 물고기, 나비, 강아지 (측면 실루엣이 잘 읽힘) |
| object | 집, 로켓, 우산, 컵, 병, 선인장, 풍선 |
| season | 크리스마스트리, 눈사람, 호박, 벚꽃, 선물상자 |
| brand | 알파벳 Z (Z-Arrows 로고), 화살표 모양 |

## 산출물

- 위 출력 형식으로 **카테고리당 4~6개, 총 30개 이상**.
- 같은 소재라도 normal용(단순·작게)과 boss용(크고 디테일) 변형을 만들어도 좋다.
- 각 실루엣 아래에 한 줄로 의도를 적어라 (예: "챕터3 보스 — 클리어 리버스 연출에서
  토끼가 드러남").

## 자가 검증 체크리스트 (제출 전 스스로 확인)

- [ ] 모든 행 길이가 동일한가
- [ ] `#` 개수가 80~200인가 (직접 세어라)
- [ ] 상하좌우 연결로 한 덩어리인가
- [ ] 1셀 폭 부분이 전체의 20% 이하인가
- [ ] 3미터 떨어져서 봐도(작게 봐도) 뭔지 알아볼 수 있는가
