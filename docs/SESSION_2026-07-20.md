## 1. 이번 세션에 한 일

### 1.1 문제은행을 앱에 연결 (핵심)

기존에 `tools/atlas/all_boards.json`의 **783개 보드**(국가 216 + 도시 567, 라인 포함·
풀림 검증됨)가 앱에 전혀 닿지 않고 있었다. 앱은 런타임에 `generateLevel`로 자체
생성 중이었고, 그래서 **도시 스테이지가 존재하지 않았다**.

- `tools/atlas/build_bank.py` 신규 — 보드 은행을 캠페인 형태로 포장해
  `assets/campaign/bank.json`(2.3MB) 생성
- `CampaignRepository`가 이 자산을 읽도록 재작성. 런타임 생성 폐기
- 도시→국가 매핑: `admin1_cities.json`(정본) + `world_campaign_order.json`(보완)
- 결과: **221 라운드 / 800 스테이지**. 도시 보유 라운드 143개

**검증**: 구운 800판 전부가 원본 보드와 **화살 배치가 정확히 일치**함을 대조 확인.
크롭은 좌표를 평행이동만 했고 퍼즐 자체는 바꾸지 않았다.

### 1.2 큰 보드 대응 — 줌 (사용자 승인됨)

칠레 138×26 / 272화살은 화면에 맞추면 셀이 4px라 한 개도 못 누른다.

- 두 손가락 핀치 줌 + 팬 (한 손가락은 탭 전용 — 탭이 게임의 전부라 드래그로 판이
  흔들리면 안 됨)
- **명시적 `+` / `−` 버튼** — 핀치를 시도조차 안 하는 플레이어가 막히지 않도록
- 헤더에 뷰 리셋 버튼
- 판 크기 자체는 **건드리지 않음**(확정 규격 유지). 사용자가 "1줌"으로 확정

### 1.3 출시 차단 버그 수정

**릴리즈 APK가 첫 프레임도 못 그리고 죽었다.** 디버그만 테스트했으면 못 잡았다.

```
Failed to create an instance of androidx.work.impl.WorkDatabase
```

`google_mobile_ads`가 끌어오는 WorkManager의 Room 구현을 R8이 제거.
`android/app/proguard-rules.pro` 신규 + `build.gradle.kts`에 연결. 릴리즈 설치·
실행·플레이 확인 완료.

### 1.4 그 외 수정

- **부팅 병목**: 광고·결제·Firebase·게임서비스 SDK를 순차로 await 하느라 로딩바가
  50%에서 10초+ 멈춰 있었다. 첫 화면에 필요한 데이터만 대기하도록 변경
- **보드 마스크 지연 생성** — 800개를 부팅 때 펼치면 수십만 레코드
- **줌 시 보드가 헤더 위로 넘침** → 게임 표면 클리핑
- **스테이지 표기를 라운드 기준으로** — "641번째"가 아니라 "3 / 12"
- **라운드 인트로의 죽은 "Paths 0"** 제거, 라운드 성격에 맞는 문구로
- **월드맵 스테이지 단위 부분 채색** + "N개국 완료" 카운터
- **설정 행 높이 통일** — Switch가 있는 행만 높았음 (사용자 지적)

## 3. ⚠️ 미완 · 사용자 지적사항 (이어받는 사람이 먼저 볼 것)

### 3.1 도시 없는 78개 라운드 — **미구현**

현재 1스테이지(국가 피날레만)다. 사용자 지시는 **`path > 국가` 2스테이지**.

- 필요한 것: **path 보드 78개 생성**(사각형 마스크, 화살 60개 이상)
- `build_bank.py`에서 도시 없는 국가의 stages를 `[path, country]`로 구성
- ⚠️ **기존 문제 자산은 절대 건드리지 말 것.** path 보드만 새로 만들어 추가한다

### 3.2 월드맵 — 사용자 평가 "노답"

구체적 요구사항 미확인. 현재 상태와 알려진 문제:

- 캠페인이 면적 오름차순이라 **초반 수십 라운드가 지도에 거의 안 보인다**(미세 섬나라).
  성취 엔진(지도 채색)이 초반에 사실상 작동하지 않음
- 현재 국가로 줌인하는 코드는 있으나, 튜토리얼 섬나라들이 168×65 점격자에 없어
  전세계 뷰로 폴백됨
- 임시로 "N개국 완료 · 221개국 중" 카운터를 넣어둔 상태
- **재설계 방향을 사용자에게 확인해야 함**

### 3.3 화살 60개 하한

사용자 지시: **"화살표 개수 최소 60개 이상의 문제로 출제해라."**

- 본 은행(도시 80~150, 국가 80~334)은 이미 충족
- **미충족: 온보딩 사다리 25판(화살 4~15)** — 캠페인 1~5라운드
- ⚠️ 이 세션에서 이걸 재생성하려다 **기존 자산을 덮어써서 되돌렸다.**
  `tools/atlas/onboarding_boards.json`과 `tool/export_onboarding.dart`는
  **원본 그대로 복구된 상태**다
- 참고(재생성 시도에서 얻은 실측): 이 섬 마스크들을 60화살까지 키우면
  통가 188×87, 케이맨 49×153처럼 판이 사방으로 흩어진다. 프랑스령 남방(14×19)과
  남조지아(27×27)만 60화살에서 형태가 유지된다
- **어떻게 처리할지 사용자 결정 필요** (사다리 제거 / 다른 마스크로 재생성 / 유지)

### 3.4 튜토리얼 라운드 문구 어긋남

`build_bank.py`가 사다리를 **판이 촘촘한 순서로 재배열**했는데(첫 판 셀 크기 확보
목적), `teaches` 라벨이 원래 순서 기준이라 어긋난다.

- 1라운드 프랑스령 남방 → `(예비)` 가 화면에 그대로 노출됨
- 5라운드 통가 → "탭해서 화살 빼기" (탭은 1라운드에 배워야 함)
- 3.3과 함께 정리해야 함

### 3.5 국가 소개문 미생성

`bank.json`의 `intro`가 비어 있어 라운드 인트로가 대체 문구를 쓴다.
기획서에 따라 "읽기를 요구하지 않는" 한 줄 수준으로 구울 것.

## 4. 작업 시 주의 (이 세션에서 실제로 사고 난 것)

1. **기존 문제 자산을 건드리지 말 것.** `all_boards.json`, `onboarding_boards.json`,
   `campaign.json` 등은 사용자가 측정·검증해 만든 결과물이다. 새 보드가 필요하면
   **추가 생성**하고, 기존 파일을 재생성하지 않는다
2. **릴리즈 빌드로도 실기기 검증할 것.** 디버그만 보면 R8 크래시를 못 잡는다
3. **추측으로 결정하지 말 것.** 이 세션에서 iOS 인증서를 추측으로 골라 프로파일을
   전부 다시 만들었고, 온보딩 규격도 추측으로 해석해 자산을 덮어썼다
