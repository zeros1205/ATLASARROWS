# 히든블록스(P4) 디자인 랭귀지 → Z-Arrows 적용 리스트 (2026-07-18)

P4 코드(`lib/app/tokens/*`) + 실화면 분석 결과. ✅=그대로 이식, 🎨=페이퍼/잉크 톤으로
리컬러해서 이식, ✏️=신규 제작 필요(P4에 없음).

## 0. 토큰 아키텍처 — ✅ 통째로 이식 (최우선)

P4는 진짜 디자인 시스템이 있다: `AppPalette`(원시 색) → `AppColors`(ThemeExtension 시맨틱)
→ `AppText`(6단 타입 스케일) → `AppRadius`/`AppSpacing`/`AppDuration`/`AppCurve`.
Z-Arrows의 ZTheme(색 7개짜리)를 이 구조로 승격.

- 타입 스케일(강제 6단): display 30/w800(-0.4) · title 22/w800 · headline 18/w700 ·
  body 16/w500 · label 14/w600 · caption 12/w600. **한글=Paperlogy, 라틴=Outfit,
  폴백=Pretendard** — 폰트 3종 P4에서 복사(한글 UI 폰트 백로그가 이걸로 해결).
  자간 음수는 Outfit에만(한글은 0). 워드마크 display 38/w800/-1.5.
- 라운드: cube3·xs4·sm8·md12·lg16·xl24·xxl32·pill999 (+카드 30/타일 26/답 22 —
  잉크 톤엔 과하게 몽글하니 **카드 16~20으로 타이트하게** 조정 권장)
- 간격 4pt 그리드(기본 갭 md 12), 모션: fast120/normal240/slow400/verySlow720/
  celebration900 + gentle(easeOutCubic)/snap/spring/smooth. **OS 리듀스모션 자동 대응
  (AppMotion) 패턴 포함 이식.**
- 배경은 **플랫 단색**(P4도 그라데 전부 제거함) + FlatCard(헤어라인 보더, 그림자 없음)
  — 우리 페이퍼+잉크 방향과 정확히 일치.

## 1. 온보딩 — ⚠️ P4 방식 채택 안 함 (사용자 확정, 2026-07-18)

**빠른 게임 시작이 최우선.** P4의 PageView 캐러셀 대신 **레벨 1~3 자체가 튜토리얼**:
- 가이드 문구를 플레이 화면에 직접 표시(보드 하단 필), 눌러야 할 화살표를
  **컬러 + 블링크로 강조**, 실제로 누르게 해서 학습.
- 스텝 설계: L1 = ①강조된 자유 화살표 탭→탈출 학습 ②막힌 화살표를 일부러 탭하게
  유도→범프+하트 차감 학습 ③나머지 자유 플레이. L2~L3 = 강조 빈도를 줄이며 서서히
  손 떼기(예: L2는 첫 수만 강조, L3는 문구만).
- 튜토리얼 중에는 강조된 화살표 외 탭 무시(오학습 방지).
- P4에서 가져올 것: 한글 줄바꿈 헬퍼 `_keepWholeWords`(U+2060), 가이드 필 타이포.
- 설정 → "게임 방법"은 추후 별도(백로그).

## 1.5 플레이 방식 / 레벨 구성 (사용자 확정, 2026-07-18)

- **4+1 사이클**: 레벨 4개는 기본 도형 쉐잎(BoardMasks rect/ellipse/diamond/blob),
  **5번째는 어려움 레벨 = 아틀라스 실루엣** 출제.
- 아틀라스 어려움 레벨은 **정보 카드**와 함께: 타이틀(예: "리비아의 영토") +
  설명부(예: "리비아의 국토 면적은 약 1,759,541 km²로 세계에서 17번째, 아프리카에서
  4번째로 큽니다.") → 재미+학습 포지셔닝, 도감(컬렉션) 기능과 시너지.
- ⚠️ 리비아는 포맷 **예시**일 뿐. 실제 출제 순서/레벨 커리큘럼은 **아틀라스 확정 후
  전면 재정비** — 그 전까지 레벨 배치 작업 금지.
- **컬러 테마는 라이트/다크 2종만 제공** (P4의 11팔레트 시스템 채택 안 함 — 토큰
  구조만 가져와 2테마로 운용).

## 2. 팝업/바텀시트 — ✅ 컴포넌트째 이식

- 중앙 다이얼로그: `showSlideDialog` — 페이드+스케일 0.94→1.0, 220ms easeOutCubic,
  배리어 #8C1A1F35. 컨테이너: surfaceElevated, radius 24, 패딩 (24,24,24,18).
  구성: title 22/w800 중앙 → 8 → body 18/w500 secondary → 24 → 버튼 행
  (취소=아웃라인 필 1.5px / 확인=액센트 필, danger=에러색, 높이 48, pill).
- 바텀시트: `ResultBottomSheet` — 아래에서 슬라이드업, 상단만 radius 28, maxWidth 520,
  배경 BackdropFilter blur 6, 드래그 해제 없음. 결과 시트 상단에 인라인 광고 슬롯.
- **전역 프레스 시그니처: AnimatedScale 0.96 (90ms) → 릴리즈 오버슈트 (220ms
  easeOutBack) + 라이트 햅틱** — 모든 탭 요소에 통일. (목업에도 이미 유사 적용됨)
- 다이얼로그 maxWidth 420 / 컨텐츠 컬럼 520 (태블릿 대응 상수).

## 3. 헤더 — 🎨 우리 확정안과 병합

- P4 허브: 좌 LifeBar 필(하트 23px #FF4D6D + 수량 14/w800 + 리필 카운트다운 MM:SS) ·
  우 GemChip 필. 필 = surface + 헤어라인 보더 + 그림자 없음, radius pill, 패딩 10×5.
- P4 플레이 헤더: 뒤로(44px 슬롯) · 중앙 STAGE n display 30 · 우측 BGM 토글(44px 슬롯
  균형). 보스 스테이지(10의 배수)에 빨간 HARD 배지 필.
- Z-Arrows 확정안 유지: **LEVEL 정중앙 + 하트는 보드 좌상단**. 가져올 것: 44px 슬롯
  균형 규칙, **하트 리필 카운트다운 붙은 LifeBar 필**(하트 경제와 결합 — 보드 좌상단
  하트를 이 필 스타일로), 보스 레벨 HARD 배지, BGM 토글 위치.

## 4. 플레이 화면 구성 — 🎨 골격 이식

P4 순서: 헤더 48px → (HARD 배지) → 콘텐츠 컬럼(패딩 24,10,24,18, maxWidth 520):
별+타이머 → 8px 진행바(액센트 그라데, 구간 눈금, 막판 빨강) → 미션 한 줄(18/w700
secondary) → 보드 → 선택지 → 제출(60px) → **하단 풀블리드 배너**.

- Z-Arrows 매핑: 헤더 → 보드(Expanded) → **부스터 바(P4엔 없음 — 우리 오리지널 유지,
  단 버튼을 FlatCard+프레스 시그니처로)** → 배너. 진행바는 사용자 결정으로 제외.
- 카운트다운 중 보드 가림막(불투명 cover + 3·2·1) — 우리도 레벨 시작 연출로 채택 검토.

## 5. 배너/광고 — ✅ 슬롯 설계 이식

- **높이 예약 패턴**: 배너 비활성이어도 50px 항상 예약 → 켜고 끌 때 리플로우 없음.
- 플레이 배너: 풀폭 anchored adaptive, 위 12px 갭(제출 버튼 오탭 가드), 자체
  bottom SafeArea, **라우트 안착 380ms 후 attach**(잰크 방지), 결과 시트 열리면
  suppress하되 높이는 유지.
- 결과 시트 상단 인라인 adaptive (최대 250dp) + **플레이 중 프리로드**(warm) → 즉시 표시.
- P4는 전면광고 없음·보상형(하트 리필)만. Z-Arrows는 전면 유지(우리 전략) + 이 슬롯
  엔지니어링만 가져옴.

## 6. 네이티브 스플래시 + 부트 플로우 — 사용자 지시: "동일하게"

- flutter_native_splash 구성 동일 구조로: 단색 플레이트(우리는 #F7F6F2) + 마크 이미지,
  android_12 블록엔 정사각 글리프만(워드마크는 OS가 크롭함), light/dark 동일 색.
  (우리는 b94407d에서 이미 유사 구성 — android12 정사각 규칙 준수 확인됨)
- **부트 플로우 이식**: BootstrapGate(첫 프레임 즉시 → 초기화하며 진행바 0→상한) →
  SplashScreen(같은 바 이어서 100% → 라우팅). 진행바: maxWidth 260, minHeight 5,
  radius 999, 캡션 "…불러오는 중 {n}%" 13/w700(테마 무관 고정색).
- ⚠️ **P4에 "개발사 화면"은 없음** — LOGAN LAND는 스플래시 에러 푸터("LOGAN LAND ©
  2026")와 설정 버전 행에만 존재. 브랜딩은 로딩 플레이트(워드마크 아트)에 접혀 있음.
  → Z-Arrows도 동일 방식 = 로딩 플레이트에 워드마크 + © 표기. 별도 타이밍 화면을
  원하면 신규 제작(✏️) 필요 — 사용자 결정 대기.
- 브랜드 상수 파일(brand.dart) 패턴 이식: nameEn/nameKo/hook/publisher "LOGAN LAND"/연도.

## 6.5 풀스크린 정책 — ⚠️ P4와 다르게 (사용자 지시, 2026-07-18)

P4는 노치 아래에서 앱을 시작(viewPadding.top+6)하지만 **Z-Arrows는 풀화면 극대화**:
- **Android**: 상단 스테이터스바 + 하단 네비게이션(제스처/독바) 숨김 —
  `SystemChrome.setEnabledSystemUIMode(SystemUiMode.immersiveSticky)`. 스와이프 시
  일시 표시 후 자동 숨김. 배너는 화면 최하단에 그대로(광고 가시성 유지).
- **iOS**: 노치/다이나믹 아일랜드 **밑선까지만** 콘텐츠 확장(펀치홀에 UI 겹치지 않게
  safe-area top은 존중하되 낭비 패딩 제거), 하단 홈 인디케이터 영역은
  `preferredScreenEdgesDeferringSystemGestures`로 최대 활용.
- 보드 영역이 그만큼 커지므로 셀 크기/탭 정확도에 직접 이득 — 줌 필요 임계도 낮아짐.

## 7. 홈/기타 — 참고 채택

- 홈: 노치 아래에서 시작(viewPadding.top+6), 인사말(닉네임 display 27 + 로테이션 문구
  20종) + 데일리 히어로 카드(액센트 솔리드, 스와이프+자동 로테이션 460ms) + 모드 카드 2장
  + **플로팅 탭바**(margin 14, h64, r32, 활성 = 52px 원 accent@0.13, 140ms).
- 화면 전환: SlideInRoute 우→좌 260/220ms 단일 규칙.
- 리더보드 리스트: 필 행 + 내 순위 액센트 솔리드 강조.
- 다크 모드: P4는 11개 팔레트 시스템 — 우리는 라이트 1개로 시작하되 토큰 구조 덕에
  다크(보류 중인 다크 네온 스킨) 추가가 공짜에 가까워짐.

## 적용 우선순위 제안

1. 토큰 아키텍처 + 폰트 3종 이식 (모든 것의 토대, 한글 UI 해결)
2. 다이얼로그/바텀시트 컴포넌트 + 프레스 시그니처 (목업 확정 UI들을 이 틀로 구현)
3. 부트 플로우(스플래시→로딩 플레이트) — 사용자 지시 사항
4. 플레이 화면 골격 + 배너 슬롯 엔지니어링
5. 온보딩 4페이지
6. 홈 리뉴얼(인사말+히어로 카드+플로팅 탭바)
