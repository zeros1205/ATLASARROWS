# Play Console 데이터 보안(Data safety) 입력값

> Google Play Console → 정책 → **앱 콘텐츠 → 데이터 보안** 설문의 정답지.
> 기준일 2026-07-24. **코드가 실제로 내보내는 데이터에 맞춰** 작성했다
> (틀리면 정책 위반이므로 SDK가 바뀌면 이 문서도 같이 고칠 것).

## 이 앱이 데이터를 내보내는 경로 (근거)

| 경로 | SDK / 파일 | 나가는 데이터 |
|---|---|---|
| **AdMob** | `google_mobile_ads` · `lib/services/ads/ads_io.dart` | 광고 ID / 기기 식별자 → 구글·광고 파트너 |
| **Sentry** | `sentry_flutter` · `--dart-define=SENTRY_DSN` | 비정상 종료 로그·진단 정보 → Sentry(제3자) |
| Play 게임즈 로그인 | `games_services` | 기기의 **기존** 구글 계정 인증(앱이 계정 생성 안 함) |
| Play 결제 | `in_app_purchase` | 결제는 **Play가** 처리. 앱은 권한 상태만 로컬 저장 |
| Firebase App Check / Storage | `firebase_*` | 기기 무결성 토큰·스탬프 다운로드 — **사용자 데이터 수집 아님** |

- **위치 아님**: 지도·영토는 정적 게임 콘텐츠지 사용자 위치가 아니다.
- **결제 내역 아님**: IAP는 Play가 처리하고 앱은 `adsRemoved`를 기기에만 저장, 전송하지 않는다.
- Sentry는 `SENTRY_DSN`이 주입된 릴리스에서만 활성(무DSN이면 no-op). 현재 배포
  워크플로가 시크릿으로 DSN을 넘기므로 **활성으로 간주**하고 선언한다. 과대 선언은
  안전하고 과소 선언이 위반이므로, DSN을 뺄 게 아니면 아래 두 유형을 유지한다.

---

## 1. 데이터 수집 및 보안

| 질문 | 답 | 근거 |
|---|---|---|
| 필수 사용자 데이터 유형을 수집/공유하나요? | **예** | AdMob·Sentry |
| 수집하는 모든 사용자 데이터를 암호화하여 전송하나요? | **예** | 전 경로 HTTPS/TLS, 평문 없음 |
| 앱에서 지원하는 계정 생성 방법은? | **앱에서 사용자가 계정을 만들도록 허용하지 않음** | 자체 회원가입 없음 |

**계정 생성 = "허용하지 않음"인 이유**: Play 게임즈 로그인·Play 결제는 기기에 이미
있는 구글 계정으로 **인증**할 뿐 새 계정을 생성하지 않는다. OAuth 등 다른 항목을 고르면
**계정 삭제 URL 제공을 강제로 요구**받는다. 자체 계정이 없으므로 삭제 URL도 불필요.

---

## 2. 데이터 유형 선택

아래 **3개만** 체크한다. 나머지(개인정보·위치·사진·연락처·금융정보·메시지 등)는 전부 미선택.

| 카테고리 | 항목 | 근거 |
|---|---|---|
| 기기 또는 기타 ID | **기기 또는 기타 ID** | AdMob 광고 ID |
| 앱 정보 및 성능 | **비정상 종료 로그** | Sentry |
| 앱 정보 및 성능 | **진단** | Sentry |

> ⚠️ **AdMob 공식 데이터 보안 안내가 광고 SDK 수집 유형의 정본이다** —
> <https://support.google.com/admob/answer/11450261>. 구글이 그 표에 "대략적 위치"
> 등을 포함해 두었으면 그쪽을 따를 것. 위 표는 SDK 구성 기준 최소 선언이다.

---

## 3. 데이터 사용 및 처리 (유형별 상세)

각 유형마다 Play가 묻는 4가지: 수집됨/공유됨 · 일시적 처리만? · 필수/선택 · 목적.

### 기기 또는 기타 ID (AdMob 광고 ID)
- 수집됨: **예**
- 공유됨: **예** (구글·광고 파트너 = 제3자)
- 처리를 위해 일시적으로만 사용? **아니요**
- 필수 여부: **필수** (기본적으로 광고 노출. 광고 제거는 유료 구매라 기본 상태 기준 필수)
- 수집·공유 목적: **광고 또는 마케팅** / 분석 / 사기 방지·보안 및 규정 준수

### 비정상 종료 로그 (Sentry)
- 수집됨: **예**
- 공유됨: **예** (Sentry = 제3자)
- 처리를 위해 일시적으로만 사용? **아니요**
- 필수 여부: **필수** (사용자가 끌 수 없음)
- 수집·공유 목적: **분석** / 앱 기능

### 진단 (Sentry)
- 수집됨: **예**
- 공유됨: **예** (Sentry)
- 처리를 위해 일시적으로만 사용? **아니요**
- 필수 여부: **필수**
- 수집·공유 목적: **분석** / 앱 기능

## 유지보수 규칙

- **AdMob·Sentry를 빼거나 새 데이터 수집 SDK(분석·위치·로그인 등)를 추가하면**
  이 문서와 Play 설문을 같이 갱신한다. Firebase Analytics를 넣는 순간 "앱 활동" 유형이
  추가된다.
- iOS ATT는 이 설문과 무관하다(App Store용). Play는 광고 ID를 항상 수집으로 본다.
