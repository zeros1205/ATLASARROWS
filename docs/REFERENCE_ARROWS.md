# 레퍼런스 분석: "Arrows" 앱 (2026-07-18 캡처)

원본: `P5_Z-ARROWS/reference/app_Arrows/` (스크린샷 7장 + 플레이 영상 103초).
동일 장르(스네이크 화살표 탭 탈출)의 라이브 서비스 앱. 우리와의 차이점 중심으로 정리.

## 1. 게임플레이 UX

| 항목 | 레퍼런스 동작 | 우리 현황 |
|---|---|---|
| 튜토리얼 | 화살표 3개 + 손가락 + "이동하려면 탭하세요" 한 줄. 텍스트 설명 없음 | 없음 (온보딩 0) |
| 실수 피드백 | 화면 전체가 연한 빨강으로 플래시 + 해당 라인이 빨갛게 유지 | 라인 범프+빨강 플래시, 쉐이크 |
| 탈출 연출 | 파란 화살표가 화면 밖까지 시원하게 날아감 | 동일 (우리도 있음) |
| 진행바 | 헤더 바로 아래 얇은 바 — 레벨 내 클리어 비율 표시 | **없음 — 채택 추천** |
| 난이도 라벨 | 상단 중앙 "어려움" 텍스트 (하트는 그 아래 중앙) | 없음 |
| 픽처 마스크 | **하트 모양 레벨 실재** (어려움 등급) | 아틀라스 방향과 일치 — 검증됨 |
| 큰 보드 줌 | **"줌을 위해 핀치하세요" 필 + 제스처 아이콘 온보딩** | 줌/팬 계획만 있음 — 검증됨 |
| 레벨 인트로 | 레벨 사이에 "Arrows / 레벨 N / 난이도 / 플레이" 전용 화면 | 없음 — 전면광고 삽입 지점으로 유용 |
| 헤더 | 좌측 ←·⟳ 붙여 배치, 중앙 난이도+하트 | 우리는 LEVEL 중앙 + 하트 보드 좌상단(사용자 확정) |

## 2. 하트/재시작 경제 (사용자 관찰 확인됨)

- 재시작(⟳·다이얼로그의 "다시 시작") 완전 무료·무제한.
- 하트 소진 → "계속하시겠습니까?" 다이얼로그, **보드 진행 유지한 채**:
  - **1회차: "무료로 생명을 채우고" + 초록 [계속 플레이]** — 첫 실패는 공짜 리필 (온보딩 배려)
  - **2회차부터: "광고를 보고 생명을 채우고" + [목숨 추가(광고 아이콘)]** — 보상형 광고 리필
  - 보조 버튼 = 다시 시작 (무료, 진행 초기화)
- 즉 하트의 상품성 = "진행 유지". 재시작을 막지 않아도 광고 리필이 팔리는 구조.
  광고 미로딩 시 버튼에 오프라인 아이콘 표시.

## 3. 수익화 구조

- ⚠️ **캡처에서 광고(배너 포함)가 전혀 안 보이는 건 캡처 기기에 광고차단 앱이 활성화돼
  있었기 때문** (사용자 확인, 2026-07-18). 배너/전면의 실제 운영 여부·위치는 이 캡처로
  판단 불가 — "무배너 전략"이라는 초기 해석은 폐기. 우리 배너 전략은 그대로 유지.
- 광고 제거 IAP **₩9,999 단품** (설정 내): "레벨 사이 전면 제거, 생명·힌트 광고는 유지" —
  보상형은 남기는 문구 벤치마킹 가치 있음.
- 부스터/아이템 UI는 미노출 (광고제거 문구에 "힌트 광고" 언급은 있음 — 힌트 기능은 존재하되
  광고차단으로 안 보였을 가능성). → 우리 부스터 바(HINT/PULL/SHIELD)는 여전히 차별점.

## 4. 메타/설정

- 다크 모드 토글 (우리 "다크 네온 스킨 보류"와 연결)
- 새로 고침 빈도(고주사율) 토글, 진동/소리 토글
- 언어 10개 (en de fr it ja ko pt-BR ru es tr) 인앱 선택
- 계정 연결: Google / Play Games 로그인으로 진행 저장
- 하단 탭: 홈 / **레벨 20 잠금 탭 / 레벨 10 잠금 탭** / 설정 — 잠금 모드를 탭으로 미리
  보여주는 티징 (해당 레벨 도달 시 언락되는 추가 모드로 추정)
- 평가하기 / 문의 / 개인정보·약관 / 구매 복원

## 5. 채택 우선순위 제안

1. 하트 경제 이식: 실패 다이얼로그 = 진행 유지 + 1회 무료 리필 → 이후 보상형 리필 (재시작은 무료 보조)
2. 진행바 (라인 클리어 비율)
3. 튜토리얼 (화살표 3개 + 한 줄 안내)
4. 레벨 인트로 화면 (전면광고 지점 겸용)
5. 실수 시 화면 연빨강 플래시
6. 핀치 줌 온보딩 필 ("줌을 위해 핀치하세요")
7. 광고 제거 단품 IAP (전면만 제거)
