# Z-ARROWS 개발 계획

> Shift the Arrows, Clear the Maze.

## 1. 레퍼런스 분석

| | Arrow Puzzle (Easybrain) | Arrows – Puzzle Escape (ecffri) |
|---|---|---|
| 핵심 규칙 | 화살표를 탭 → 가리키는 방향의 경로가 비어 있으면 화면 밖으로 탈출. 막히면 이동 불가. 보드를 전부 비우면 클리어 | 동일 계열 (tap-away 장르) |
| 특징 | 타이머 없음, 릴랙싱, 데일리 챌린지 + 월간 트로피, 힌트 | 미니멀 디자인 |
| 성공 요인 | 규칙 3초 이해 + 손맛(사운드/햅틱) + 짧은 세션 | 〃 |

**장르 본질**: "정리하는 쾌감". 규칙은 극단적으로 단순하고, 리텐션은 *촉감(juice)* 과 *데일리 루프*가 만든다.

## 2. 게임 규칙 (코어)

> 화살표는 타일이 아니라 **그리드 셀들을 따라 꺾이며 이어지는 선(스네이크)**.
> 수십 개의 선이 얽혀 하나의 미로를 이룬다.

- 각 화살표 = 셀 경로(폴리라인) + 머리(화살촉) 방향.
- 탭 → 선 전체가 **자기 경로를 따라 기차처럼 미끄러지며** 머리 방향 직선으로 화면 밖 탈출:
  - 머리의 직선 진행 경로가 비어 있으면 탈출 (제거). 자기 몸통은 막지 않음(셀프 크로스 허용 — 나선 모양 가능).
  - 다른 선에 막히면 **막힌 지점까지 밀려갔다 튕겨 복귀 + 하트 1개 차감**.
- **하트 3개**: 실수 3번이면 레벨 실패 → 재시도. (레퍼런스 검증된 긴장 장치)
- 모든 선을 내보내면 클리어. 타이머 없음.

### 픽처 레벨 (콘텐츠 차별화의 핵심)
선들의 실루엣이 **그림**(하트, 토끼, 별, 숫자…)을 이루는 레벨. 클리어 후 "이 레벨은 ♥였다"를
보여주는 리버스 연출 → 수집/공유 욕구. 챕터 보스 레벨로 배치.

### 특수 요소 (챕터 진행에 따라 단계 해금)
1. **상자(크레이트)** — 경로를 막는 고정 장애물, 특정 선이 탈출해야 파괴 (Lv 20~)
2. **이중 머리 화살표** — 양 끝이 머리, 탭한 쪽으로 탈출 (Lv 40~)
3. **얼음 선** — 인접 선이 탈출할 때 해동 (Lv 60~)
4. **색 게이트** — 같은 색 선만 통과 (Lv 80~)
5. **포탈 쌍** — 선이 통과하면 반대편으로 (Lv 100~)

## 3. 차별화: "심플하지만 강렬하게" (후킹 3요소)

### 3-1. 비주얼
- **다크 배경 + 네온 형광 화살표** (Easybrain의 밝은 파스텔과 정반대 포지션).
- 화살표는 방향별 색상 구분 옵션 (색약 모드 포함).
- 브랜드 모티프 "Z": 번개 같은 지그재그 궤적, 로고/스플래시에 일관 적용.

### 3-2. 애니메이션 (juice 패스 — 별도 마일스톤으로 확보)
- **탈출**: ease-in 가속 + 모션 트레일(잔상) + 이탈 지점 가장자리 글로우 + 파티클 버스트.
- **막힘**: squash & stretch bump + 부딪힌 타일 미세 흔들림 전파.
- **콤보**: 3연속부터 화면 미세 줌 펀치, 5연속 슬로모 순간 연출.
- **클리어**: 마지막 화살표 슬로모 → 컨페티 → 보드 붕괴 → 결과 카드.

### 3-3. 사운드 & 햅틱
- 탈출음: ASMR 계열 "휙-팝". **콤보마다 반음씩 피치 상승** (도파민 루프의 핵심).
- 막힘음: 낮고 짧은 "툭" + 가벼운 햅틱.
- 클리어: 상승 아르페지오 + 강한 햅틱 패턴.
- 배경: 무음에 가까운 앰비언트 (사운드가 아니라 효과음이 주인공).

## 4. 레벨 설계

- **역삽입 생성기**: 빈 보드에 선을 하나씩 삽입하되, 각 선의 탈출 직선 경로가 *먼저 삽입된
  선들*만 피하面 됨 → 역순 제거가 항상 가능 = **풀 수 있는 레벨 보장**. 스네이크 카빙으로
  영역(실루엣 마스크 = 픽처 레벨)을 채우는 방식과 결합.
- 난이도 지표: 보드 크기, 선 개수/길이, 강제 순서 의존성 깊이, 셀프 크로스/나선 빈도.
- 생성기 출력 + 수작업 튜닝 혼합. 레벨은 JSON 에셋으로 번들.

### 레벨 볼륨 전략
- 생성기가 완성되면 레벨 양산 비용은 사실상 0 (생성 → 솔버 검증 → 난이도 지표 자동 등급).
- **런칭 시 1,000개** (챕터 50개 × 20레벨): 초반 1~3챕터는 전수 수작업 튜닝, 이후는 생성기 + 샘플링 검수.
- **무한 모드(Endless)**: 스테이지를 다 깬 유저용 — 시드 기반 온디바이스 실시간 생성이라 콘텐츠 고갈이 없음. 번들 용량도 무관.
- 라이브옵스로 챕터 단위 추가 (Remote Config로 원격 배포 가능 구조).
- **데일리 챌린지**: 날짜 시드 기반 생성 + 월간 달력 트로피 (레퍼런스 검증된 리텐션 장치).
- 힌트: 다음 탈출 가능 화살표 하이라이트 (보상형 광고 연동).

## 5. 기술 스택 — P4(히든블록스) 재사용

| 영역 | 선택 | 비고 |
|---|---|---|
| 엔진 | Flutter + Flame | P4 동일. 그리드/탭 게임에 충분 |
| 상태 | flutter_riverpod | 수동 프로바이더 |
| 저장 | hive | 진행도/설정 로컬 |
| 백엔드 | Firebase (Analytics, Remote Config, Crashlytics) | P4 핀 버전 참고 (2026-07 firebase 호환성 이슈 주의) |
| 광고 | AdMob 배너/전면/보상형 | 전면 빈도는 Remote Config로 |
| IAP | 광고 제거 + 힌트팩 | P4 IAP 코드 재사용 |
| 리더보드/업적 | games_services v5 | Flutter ≥3.44 필요 (P4 pubspec 참고) |
| CI/CD | GitHub Actions (P4 워크플로 복사) | iOS 버전 규칙: pubspec version 항상 스토어 이상 유지 |

## 6. 수익화

- 전면 광고: N레벨 클리어마다 (초반 10레벨 노출 금지, N은 Remote Config).
- 보상형: 힌트 / 실행취소(undo) / 데일리 재도전.
- 배너: 게임플레이 하단 (성과 보고 제거 여부 판단).
- IAP: 광고 제거(₩5,500 내외), 힌트 번들.

## 7. 마일스톤 (주 단위)

| 주차 | 목표 | 완료 기준 |
|---|---|---|
| M1 | 코어 프로토타입 | 그리드 렌더, 탭 → 이동/충돌/탈출, 클리어 판정. 하드코딩 레벨 5개로 폰에서 플레이 가능 |
| M2 | **Juice 패스** | §3의 애니메이션/사운드/햅틱/콤보 전부. "만지는 맛" 기준으로 통과 판정 |
| M3 | 레벨 시스템 | 역방향 생성기 + 검증 솔버 + 난이도 자동 등급, **레벨 1,000개** (초반 3챕터 수작업 튜닝), 무한 모드, 특수 타일 2종 |
| M4 | 메타 & UI | 홈, 레벨맵, 데일리 챌린지, 설정(사운드/햅틱/색약), ko/en 로컬라이징 |
| M5 | 수익화 & 출시 준비 | 광고/IAP/Analytics 이벤트, 스토어 자산(아이콘/스샷/영상), 내부 테스트 트랙 업로드 |
| M6~ | 소프트런칭 & 라이브옵스 | 지표(D1/D7, 레벨 이탈) 보고 난이도 튜닝, 특수 타일 3~5 추가, 챕터 단위 콘텐츠 증설(2,000+) |

**M2를 M3보다 앞에 둔 이유**: 이 장르의 승부처는 콘텐츠 양이 아니라 손맛. 손맛이 안 나오면 레벨 100개는 의미 없음 — 프로토타입 단계에서 juice로 먼저 검증한다.
