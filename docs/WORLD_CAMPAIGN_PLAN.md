# 세계지도 탐험 캠페인 기획 (2026-07-18)

플레이 컨셉 재정립: **도형 퍼즐 → "세계지도를 채워나가는 여행"**. 캠페인의 척추가
세계지도이고, 각 국가가 챕터, 그 국가의 주요 도시들이 레벨이다. 도시의 실제 행정경계
실루엣을 퍼즐 보드 마스크로 쓴다. 클리어하면 도감(세계지도 모으기)에 수집된다.

> 근거 데이터: `tools/atlas/plan_world_campaign.py` 실행 결과. Natural Earth 50m 국가
> 폴리곤(구면 면적) + populated places(도시·인구). 산출물 `world_campaign_order.json`.

## 1. 핵심 컨셉 — World Map Exploration

- 척추 = **세계지도**(페이퍼+잉크 아트로 그린 점묘 세계지도). 잠긴 국가 = 흐린 점,
  클리어한 국가 = 잉크로 채워진 실루엣. **플레이할수록 세계가 그려진다** — 우리 아트
  아이덴티티(종이·잉크)와 완벽히 맞물리는 진행감.
- 국가 = 챕터. 그 국가의 도시들 = 레벨. 국가 완주 = 여권 스탬프 + 잉크 채색 + 정보 카드.
- 대륙별 그룹핑으로 지도 네비게이션. 도감(세계지도 모으기) = 이 캠페인의 수집 레이어.

## 2. 난이도 램프 — 국가 면적 오름차순 (데이터 검증 완료)

사용자 지시("작은 국가 우선, 미국 등 후반부")가 **면적 오름차순 정렬 하나로 자동 충족**됨:

| | 면적 | 도시 수(NE) | 성격 |
|---|---|---|---|
| 초반 | 바티칸 1 · 투발루 10 · 모나코 12 · 몰디브 66 · 산마리노 68 km² | 0~1 | 초소형, 단순 |
| … | … | … | 점증 |
| 후반 | 인도 71 · 호주 36 · 브라질 46 · 중국 98 · **미국 111** · 캐나다 45 · 러시아 82 | 다수 | 거대·복잡 |

- **미국(111 도시)·중국(98)·러시아(82)·인도(71)** 는 면적도 크고 도시도 많아 자연히
  후반 "보스 챕터"가 된다 — 지시와 정확히 일치.
- 국가 내 도시 순서 = 마스크 셀 수 오름차순(작은 도시 → 큰 메트로). 전역 스무딩 1패스로
  국가 경계 난이도 점프 완화.

## 3. 국가별 레벨 수 = 국가 규모에 비례 (5개 고정이 아니라 티어링)

⚠️ **데이터 현실**: NE 기준 도시 5개 이상 보유국은 241개 중 **53개뿐**(≥3: 86, ≥1: 204).
초소형국은 도시가 0~1개. 따라서 "국가당 5개 고정"은 불가 → **규모별 티어링**:

| 티어 | 국가 예 | 레벨(도시) 수 |
|---|---|---|
| T0 초소형(도시국가) | 모나코·산마리노·싱가포르 | 1 (도시=국가) |
| T1 소형 | 대부분 섬·소국 | 2~3 |
| T2 중형 | 표준 국가 | 4~5 |
| T3 대형 | 스페인·이집트·아르헨티나 | 6~8 |
| T4 거대(보스 챕터) | 미국·중국·러시아·인도·브라질·캐나다·호주 | 8~12 |

- "주요 도시 5개 이상"은 T2+ 국가의 목표치로 유지, 소국은 있는 만큼.
- **최종 제작 가능 레벨 수** (`estimate_levels.py`, 감쇠 = OSM경계 82% × 마스크하한 90% = 74%):

| 시나리오 | 원시 | 감쇠 후(최종) |
|---|---|---|
| top-5 고정, NE 데이터 | 519 | ~383 |
| **티어링, NE 데이터 (기준안)** | 553 | **~408** |
| 티어링, 풍부한 DB(GeoNames) 천장 | 1,005 | **~742** |

  → **현실 기준안 ≈ 400 스테이지**(도시국~거대국 티어링, 현 NE 도시 DB + 정직한 감쇠).
  더 풍부한 도시 DB(GeoNames 등)로 티어 상한을 꽉 채우면 **최대 ~740**. 204개국,
  거대 보스 챕터 11개(미국·중국·러시아·캐나다·브라질·호주·인도·아르헨티나·카자흐·알제리·그린란드).
  감쇠 15~20%는 재시도/대체 페치로 일부 회복 가능(+50~100). 튜토리얼 3레벨·연습 모드는 별도.

## 4. 데이터 파이프라인 (2소스)

1. **선정·순서 = Natural Earth populated places** (인구·국가 태그). 국가별 인구 상위 K개 픽.
2. **마스크 모양 = OSM Nominatim 행정경계 폴리곤** → 기존 `raster_cities.py`로 격자화.
   - ⚠️ 최대 리스크: 경계 폴리곤 **미검색 ~15~20%**(뭄바이·요하네스버그 등 이미 확인).
     500~1000 도시 벌크 페치 시 실패분 발생 → 대체 규칙(행정레벨 조정 / city-proper↔metro /
     소수 수작업). 성공분만 레벨화, 목표 미달국은 레벨 수 축소.
   - 벌크 페치: 5도시×200국 ≈ 1000 req × 1.1s = **~18분(1회, 캐시)**. 실현 가능.
3. **정보 카드 텍스트**: 국가(면적·순위)는 자동 산출. 도시(인구·특징)는 인구는 NE에서
   자동, 나머지 특징 문구는 **저작 필요**(콘텐츠 비용) — 템플릿+선별 저작.
4. **크로스플랫폼**: 캠페인은 **레벨(마스크+라인) 전체를 JSON 프리베이크**. 마스크는
   지오데이터라 결정적이나 라인 생성은 dart:math Random(VM/web 상이) → 프리베이크로 고정.

## 5. 기본 도형 / 4+1 사이클과의 관계 — 결정 필요

세계지도 캠페인이 기존 "4+1(기본 도형 4 + 아틀라스 1)"을 대체하는가?
- **(A) 순수 도시 캠페인(추천)**: 캠페인 전부가 실제 도시 레벨. 기본 도형은 튜토리얼
  (L1~3)과 별도 "연습/무한 모드"로. 모든 레벨이 실존 장소 = 컨셉 순수·수집 동기 강함.
- (B) 하이브리드: 국가 챕터 안에서 기본 도형 3~4개 워밍업 + 그 나라 대표 도시 1개. 4+1
  리듬 유지하나 컨셉이 희석.
- 권장 = **(A)**. 4+1은 이 컨셉으로 대체(기본 도형은 튜토리얼/연습으로 강등).

## 6. 도감·메타 통합 (이미 확정된 도감과 결합)

- 세계지도 모으기 도감 = 이 캠페인의 수집 화면 그 자체. 클리어 국가 잉크 채색 + 정보 카드.
- 도시 클리어 → "도감에 수집됨" 배지. 국가 완주 → 여권 스탬프 + 대륙 진척.
- 동물 도감은 별도 트랙(이벤트/보너스 레벨 or 별도 챕터)으로 병행.

## 7. 열린 이슈 / 결정 대기

1. 4+1 대체 여부 = (A)/(B) — **6.5절 결정 필요** (추천 A)
2. 초소형국(도시 0~1, 마스크 80셀 미만) 처리: 국가 실루엣으로 대체 / 병합 / 생략
3. 캠페인 규모 목표: 500(top-5) vs 700(거대국 확장) vs 1000(top-K 심화)
4. 도시 정보 카드 저작 범위(전 도시 vs 대표 도시만)
5. 동물 도감 배치(이벤트 vs 챕터)

## 8. 빌드 단계

- **P1 데이터 확정**: 국가 순서(면적) + 티어별 도시 선정 리스트 (일부 완료 —
  `world_campaign_order.json`). 초소형국 규칙 확정.
- **P2 벌크 자산**: OSM 경계 벌크 페치 → 마스크 격자화 → 실패분 대체. 도시 마스크 아틀라스.
- **P3 난이도·순서**: 마스크 셀 수 기반 국가내·전역 정렬 + 스무딩.
- **P4 프리베이크**: 레벨(마스크+라인) JSON 생성 + 검증(솔버).
- **P5 통합**: LevelRepository 연결 + 세계지도 UI(잉크 채색 지도) + 정보 카드/도감.
